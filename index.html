  5. сделать сайт хостингом ✔

Для работы с git использовались следующие команды:

  • git init - создание локального репозитория в папке
  • git add . - добавление всех новых файлов в отслеживание
  • git commit -m"commit text" - создание комментария к изменениям
  • git remote add origin ссылка_на_репозиторий - соединение локального репозитория с удаленным
  • git push -u origin master - загрузка изменений на удаленный репозиторий
  • git checkout -b gh-pages - создание новой ветки gh-pages, которая является хостингом
  • git checkout master - возвращение на ветку master
  • git push origin master:gh-pages - загрузка всех файлов из ветки msater в ветку gh-pages, тем самым файлы заливаются на хостинг
